# Report-Ready Attack Scenario

The selected weakness is the generation of a session token with `java.util.Random` inside `Login.generateSessionToken()` (`Login.java` 183-188). In the observed flow, this token is created immediately after successful credential verification and then stored in `SharedPreferences` by `createSession()` (`Login.java` 174-176). A realistic attacker for this APK is not an internet-scale remote adversary, but a local-analysis attacker on an emulator, rooted tutorial device, or instrumented test environment who can estimate when the victim logged in and inspect or test token state. Under those conditions, the attacker can reproduce candidate outputs for nearby PRNG seeds and compare them against the generated session token or token-dependent behavior. This does not justify claiming a guaranteed remote bypass in the current build, but it does demonstrate that the app creates authentication-related state with a predictable PRNG rather than a cryptographically secure generator.
